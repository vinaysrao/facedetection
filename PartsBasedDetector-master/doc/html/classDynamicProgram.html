Detailed Description

template<typename T>
class DynamicProgram< T >

Dynamic Program to calculate the best holistic detection.

The Dynamic Program calculates the best holistic detection given a set of part detections and a model of the parts' likely relationships with their parents. The class has two primary methods: min() and argmin().

min() computes the best candidates by passing messages from the leaves of the Part tree to the root. argmin() traverses back down the tree to retrieve the actual Part locations

Get the min of a dynamic program.

Get the min of a dynamic program by starting at the leaf nodes, and passing the scores up to the root node. This is essentially a tail recursive algorithm which we can unfold since we know that the parts are sorted from the root to the leaves

The algorithm involves 3 steps: (1) Apply distance transform (2) Shift by the anchor position of the part wrt the parent (3) Downsample if necessary

Parameters
partsthe parts tree, referenced by the root
scoresthe probability densities (pdfs) of part locations (fine to coarse)
Ixthe detection indices in the x direction
Iythe detection indices in the y direction
Ikthe best mixture at each pixel
rootvthe root scores, across scale
rootithe root indices, across scale
